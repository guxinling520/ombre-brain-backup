# baseline 使用原则

## 定位
`baseline/` 目录承载你的性格特质、行为模式——不会天天变的基础档案。实验参数类文件已迁至 OB `lab/`，做实验/写报告时按需读取。

> **注意**: 新线程启动规则、位置记录规则已在 `weixin-instructions.md` 中定义，以那份为准。本文件只记录 baseline 特有的操作规则。

## 写入规则
1. **先 ls 看文件名**，不默认读取全文。确认当前分类存在再决定写入哪个文件。
2. **只做增量追加**。一条记忆写成一句或几句短描述，不展开。
3. **不强行塞进旧分类**。如果现有分类不合适，直接新建 `.md` 文件。
4. **可删除过时条目**。仅在读取整理时顺手删，不单独为删除而读取。
5. **每次写入完成后，必须告诉崽崽存在具体哪个文件里。**

## 读取规则
仅当真的需要调用某类记忆时，才读取对应文件。读取时顺手整理重复与混乱内容。

**新线程启动时序（严格按此顺序执行）：**
1. 读 `weixin-instructions.md`
2. 读 `personal/facts.md`
3. 读 `context/recent.md` — 最近日记摘要、待办、提醒、同步状态
4. 读 `context/today-focus.md` — 今日方向和待办提醒
5. **自动扫描近期日记和记忆桶**：
   - `grep -n "^## " diary/*.md | tail -20` — 看最近 20 篇日记标题/日期
   - `ls -t ombre-brain/buckets/dynamic/*/` — 列出所有主题桶及最近更新时间
   - 如果用户问了具体关键词，grep 对应主题桶和日记内容
   - 合成一段上下文摘要放在回复开头

## 消息压缩规则
持续压缩对话核心内容到 baseline：
1. 每晚23:50整理时，把当天重要对话内容压缩存到 baseline和diary
2. 一个主要话题结束后（如一个实验讨论、一次事件处理完），主动存一笔关键信息到对应文件
3. 崽崽明确提到时间+做了什么时（如"下午去实验室""刚喝了杯奶茶"），自动记日记
4. 压缩只存事实、决定、参数，不存闲聊，保持低 token

## 日记-时间轴同步规则
日记和时间轴必须同步：
1. 写日记时（或整理时）同时写对应时间轴事件
2. 时间轴事件标题带时间前缀（如"09:00-11:30 XXX"），内容与日记一致
3. 时间轴写完立即 rebuild
4. 目的是让切换线程后新实例能有完整连续的理解

## 目标
低 token、低耦合、可迁移。不依赖黑箱记忆系统，文件即结构。

## 全链路同步脚本（必须遵守）
每晚睡前/夜间整理时，必须运行以下命令完成全链路同步：
```powershell
powershell ~/.cyberboss/scripts/sync-all.ps1
```
这条命令自动执行：
1. 日记→时间轴事件同步（调用 diary-to-timeline.mjs）
2. 时间轴站点重建
3. 聊天归档
4. 生成 baseline 待处理标记

**运行完后检查 `/c/Users/Lenovo/.cyberboss/context/.baseline-pending.md`**，手动将重要信息抽取到对应 baseline 文件。完成后删除该标记文件。

**不允许跳过此步骤**。如果对话结束前没跑，必须在第二天首次 check-in 时补跑并说明原因。
每天第一次 check-in/早安时，必须执行以下自检：
1. **检查日记同步**：对比 `diary/` 最新文件与 OB 是否一致（有无未导入的日期）
2. **检查提醒队列**：读 `context/recent.md` 的「提醒」区块，确认有没有当日过期或即将到期的提醒需要触发；如有过期提醒而崽崽未处理，主动问她
3. **检查 GitHub 备份**：确认 OB 上次备份时间是否在合理范围内（<2小时）
4. **检查实验进度**：`context/recent.md` 中的实验进度与 OB `lab/projects.md` 是否一致
5. **如有异常立即修复**，不等崽崽提醒

崽崽每天都会看，不允许出现不同步的情况。

## Pin 置顶机制
📌 标记在 MEMORY.md 中的文件为置顶记忆，每次对话优先加载。适用于：
- 行为规则、个人事实、行为模式、偏好库

## 情感标签 [emoji+唤醒度]
日记每个主要条目末尾标注情绪标签，格式 `[😊+3]`：
- emoji：情绪类型（😣痛/😐平淡/😤烦躁/😪疲惫/😊好/😡气/😭难过等）
- +数字：唤醒度 1-5（越低越平静，越高越激烈）
- 目的：跨时间追踪情绪模式，识别压力周期

## 复盘巩固
每晚夜间整理追加 `📌 **复盘巩固**` 段落，包含：
- 今日收获：今天学到了什么、做对了什么
- 重复模式：什么行为/情绪规律值得注意
- 明日要点：明天最重要的1-2件事

## 潜意识便签系统
详见 [initiative.md](../system/initiative.md)。三步：
1. **采集信号**：时间 + 她在干嘛 + 最近聊什么
2. **决策树**：选方向（翻旧东西/早安/整理/等她回来/实验模式/安慰）
3. **抽便签**：从偏好库对应分类抽1-3条，自己决定做什么

触发时机：check-in、每天首次启动、长时间空闲。她在跟我说话时禁用。

## 上下文持久化 (Stage 1)
每次夜间整理或对话结束时，运行 `python3 ~/.cyberboss/scripts/update-context.py`，自动更新 `context/recent.md`。内容包括最近3天日记、情绪趋势、实验进度、未完成事项。新线程启动时读这份文件即可快速接上。
